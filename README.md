# Data Workflow Re-Engineering Case Study

## Overview
A complete process re-engineering case study showing 
how a broken data pipeline was diagnosed and redesigned 
to cut reporting time from 72 hours to same day delivery.

## What this case study covers
- AS-IS current state mapping (broken pipeline)
- Gap analysis with 6 prioritised problems
- TO-BE future state design (optimised pipeline)
- Measurable results and outcomes

## Key Findings (Gap Analysis)
- Manual Excel-based extraction causing 3 day delays
- Multiple analysts transforming same data differently
- No data lineage tracking
- 72hr reporting turnaround
- Zero self-service capability
- Governance checks done post-delivery

## Results & Outcomes
- 18% faster reporting cycles
- 72hr turnaround → same day delivery
- 40% reduction in manual analyst work
- Single source of truth established
- Zero compliance blocks
- Self-service analytics adopted

## Tools Used
- Miro — process mapping and visual design
- SQL — data flow context
- Power BI — dashboard layer in TO-BE design
